1º Ano

1º Semestre

FP
Fundamentos de Programação
Esta cadeira é relativamente simples, aprendem Python e outros básicos da programação.

Dicas para os projetos:

► Corrijam os projetos dos vossos colegas com calma de forma a obterem uma boa nota em relação à vossa avaliação e, consequentemente, a nota do vosso projeto não ser afetada negativamente

► Façam sempre um menu de boas vindas para o utilizador como aspeto "inovador" a valorizar na hora da avaliação por parte dos vossos colegas

► Não desvalorizem os projetos. Mesmo que cada projeto apenas valha 1 valor na vossa nota final, quando for a altura de fazer o APF (Avaliação Prática Final), se essa componente não estiver acima de 6.5 valores estão reprovados, e muitas vezes estes projetos compensam pelo desastre que os testes do Code Check são...

► Dediquem tempo aos projetos e corrijam quaisquer "loose ends" que estejam por aí metidas.
ITW
Introdução às Tecnologias Web
Nesta cadeira aprendem principalmente HTML, CSS e JavaScript

Normalmente, se atingirem a nota mínima na componente teórica, o professor costuma facilitar na aprovação à cadeira
  • HTML Visualizer ➡ https://codebeautify.org/htmlviewer
Dicas para os testes teóricos:

► Responder aos seguintes quizzes:
  • HTML ➡ https://www.w3schools.com/html/html_quiz.asp
  • CSS ➡ https://www.w3schools.com/css/css_quiz.asp
  • JavaScript ➡ https://www.w3schools.com/js/js_quiz.asp
  • Bootstrap ➡ https://www.w3schools.com/bootstrap/bootstrap_quiz.asp
  • jQuery ➡ https://www.w3schools.com/jquery/jquery_quiz.asp

As perguntas teóricas são extremamente parecidas a estas (pelo menos 50% do teste), a parte escrita já é um pouco questão de sorte.
Em relação ao projeto usem AI se precisarem (muito provável), no entanto garantam que percebem o que está a acontecer e deixem o mais simples possível uma vez que vão ter de apresentar o projeto.
MAS
Modelação e Análise de Sistemas
Nesta cadeira aprendem principalmente todo o processo de preparação e planeamento de um projeto ou sistema.
Geralmente costuma-se achar a cadeira um pouco chata e inútil, mas irá vir de encontro com uma futura cadeira POO (Programação Orientada a Objetos) em que será necessário ter uma perceção básica de MAS.

Dicas para o Exame Teórico:

► Estudar um pouco os slides e o material recomendado pelo professor (geralmente na última aula teórica).

► Ver Exame Teórico 23/24 ➡ https://tinyurl.com/ynntaz77
ALGA - A
Álgebra Linear e Geometria Analítica - A
Esta cadeira é de forma geral acessível (principalmente no 1ºTeste onde se consegue tirar 18+ facilmente), depois o 2ºTeste já complica bastante.
Esta cadeira terá utilidade mais à frente no 2ºAno onde aprenderão a liguagem MatLab que consiste na utilização de matrizes

Dicas:

► Realmente aproveitar o 1ºTeste

► Resolver exercícios das fichas e comparar os resultados com resoluções de alguma drive (normalmente estão mais completas do que as soluções dos professores)
C I - C
Cálculo I - C
Cadeira bastante trabalhosa mas não passa de matemática do secundário só que num ritmo e complexidade mais elevado.
Geralmente o 2ºTeste é mais fácil uma vez que um dos temas abordados é EDOs (Equações Diferenciais Ordinárias) e que seguem uma resolução padronizada.

Dicas:

► Resolver exercícios das fichas e comparar os resultados com resoluções de alguma drive (normalmente estão mais completas do que as soluções dos professores)

► Estudar em articulação com estes links:
  • Integral Calculator ➡ https://www.integral-calculator.com
  • MathDF ➡ https://mathdf.com/dif/

2º Semestre

POO
Programação Orientada a Objetos
Nesta cadeira aprendem essencialmente Java e, de forma inconsciente, articulam os conhecimentos de uma cadeira anterior, MAS.
Esta cadeira é de certa forma acessível, caso se faça o acompanhamento regular durante o semestre no entanto a entrega de guiões semanais é demasiado trabalhosa.

Dicas:

► Tentar realmente aprender Java nesta cadeira e não mais à frente no curso

► Se receberem uma avaliação discriminada sobre onde erraram (por exemplo omissão de métodos @Override que, por convenção, se devem logo definir) tentem corrigir logo para que no Exame Prático não vos falhem esses pequenos detalhes

Dicas para o Exame Prático:

► Voltar a refazer os exercícios do guiões onde vos é apresentado um sistema mais complexo, tipo venda de bilhetes, organização de um estabelecimento, etc... (uma vez que é num exercício desses que o exame se baseia)
MSF
Modelação de Sistemas Físicos
Nesta cadeira aprendem conceitos de física (mais baseados em matemática, como por exemplo métodos numéricos) e também a analisar resultados físicos com opinião crítica.
Em geral esta cadeira não é muito complicada, no entanto as aulas teóricas fazem parecer pois estão cheios de fórmulas.
Se conseguirem entender os exercícios dos guiões práticos e os exercícios que o professor disponibiliza conseguem passar de forma tranquila.

Dicas para os Testes:

► Fazer os testes de anos anteriores (são 50% iguais, muitas vezes só trocam os valores)
IAC
Introdução à Arquitetura de Computadores
Esta cadeira fala da arquitetura de computadores ao nível mais baixo possível, binário, hexadecimal, portas lógicas, etc...
O primeiro teste é definitivamente o mais fácil (consegue-se tirar 17+ facilmente), sendo que após a introdução da matéria dos Multiplexers, a matéria dificulta bastante.

Dicas:

► Realmente aproveitar o 1ºTeste

► Na época de exames, mesmo que sintam que não percebem nada de Assembly, refaçam os guiões das aulas práticas que em 3-5 dias ficam a perceber bem
MD
Matemática Discreta
Nesta cadeira falam de lógica (conceitos de Filosofia), probabilidades, funções geradoras e grafos
É uma cadeira bastante trabalhosa e um pouco difícil de entender alguns conceitos

Dicas:

► Aproveitar os minitestes (menos matérias e ainda valem alguma coisa)

► Resolver exercícios das fichas e comparar os resultados com resoluções de alguma drive (normalmente estão mais completas do que as soluções dos professores)
C II - C
Cálculo II - C
Cadeira bastante trabalhosa mas não passa de matemática do secundário só que num ritmo e complexidade mais elevado.
Geralmente o 1ºTeste é mais fácil uma vez que a maioria da matéria segue uma resolução padronizada.

Dicas:

► Resolver exercícios das fichas e comparar os resultados com resoluções de alguma drive (normalmente estão mais completas do que as soluções dos professores)

► Estudar em articulação com este link:
  • Geogebra 3D ➡ https://www.geogebra.org/3d
